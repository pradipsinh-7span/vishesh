# Columns

CSS columns are like a way to organize your content into multiple vertical sections, similar to how text appears in columns within a newspaper or magazine.

You can control the number of columns, the width of each column, and the gaps between them. When you change the number of columns or their width, the content adjusts to maintain a neat and organized appearance.

It's as if you're rearranging items on a shelf, and they automatically adapt to keep everything well-organized on your webpage.

<Callout type="info">
  <em>Make sure to give a fix or predictable height.</em>
</Callout>

<br />

## Setup

Columns include five utility classes named count, width, gap, fill and divider. The _count_ class generates values based on a number range, while the _divider_ class's width, color, and column gap settings are derived from the default Tailwind configuration but have been extended and customized by your theme.

<details>
<summary>Tailwind CSS config file setup</summary>

```js filename="tailwind.config.js"
export default {
  // ...
  theme: {
    //...
    extend: {
      // ...

      /* defaults values are there */
      /* no need to add */
      /* just for reference */
      columns: ({ theme }) => {
        // https://tailwindcss.com/docs/theme#referencing-other-values
        return {
          // special property
          // generates classes from the range
          count: "1-12",
          // column max widths
          width: {
            auto: "auto",
            "3xs": 256,
            "2xs": 288,
            xs: 320,
            sm: 384,
            md: 448,
            lg: 512,
            xl: 576,
            "2xl": 672,
            "3xl": 768,
            "4xl": 896,
            "5xl": 1024,
            "6xl": 1152,
            "7xl": 1280,
          },
          divider: {
            width: {
              // Tailwind default border widths
              // extended by your configuration file merge
              ...theme("borderWidth"),
              px: "1px",
              medium: "medium",
              thick: "thick",
              thin: "thin",
            },
            // Tailwind default colors
            // extended by your configuration file merge
            color: theme("colors"),
            style: {
              dashed: "dashed",
              dotted: "dotted",
              double: "double",
              groove: "groove",
              hidden: "hidden",
              inset: "inset",
              none: "none",
              outset: "outset",
              ridge: "ridge",
              solid: "solid",
            },
          },
          gap: {
            // Tailwind default spacing
            // extended by your configuration file merge
            // i.e., 0, px, 0.5, 1, 2, 4, 8, ..., 40, ..., 96
            ...theme("spacing"),
            normal: "normal",
          },
        };
      },
      // ...
    },
  },
  // ...
};
```

</details>

## Classes

### Base

> It is required for other properties to take effect.

<details>
<summary>columns</summary>

{/* prettier-ignore */}
```css
.columns {
  columns:
    var(--vishesh-columns-width, auto)
    var(--vishesh-columns-count, auto);
  column-rule:
    var(--vishesh-columns-divider-width, 1px)
    var(--vishesh-columns-divider-style, solid)
    var(--vishesh-columns-divider-color, transparent);
}

.columns * {
  column-span: "none";
}
```

</details>

### Column Count

> It specify the number of column count.

<details>
<summary>cols-x</summary>

> <em>there's a special property called the `count` that</em>
> <em> allows you to create classes based on a specified</em>
> <em> numeric range.</em>

> <em>By default, it covers the range from _1-12_, generating</em>
> <em> classes in the format `cols-x`, where x can be any</em>
> <em> numbers within that range.</em>

```css
.cols-x {
  --vishesh-columns-count: x;
}
```

</details>

<details>
<summary>cols-[x]</summary>

> <em>Column count has arbitrary value support.</em>

```css
.cols-\[x\] {
  --vishesh-columns-count: x;
}
```

</details>

<br />

### Column Width

> It establishes the ideal width for columns, enabling automatic column count calculation and equal space distribution within a container.

<details>
<summary>cols-w-x</summary>

```css
.cols-w-x {
  --vishesh-columns-width: x;
}
```

| class       | value  |
| :---------- | :----- |
| cols-w-auto | auto   |
| cols-w-3xs  | 256px  |
| cols-w-2xs  | 288px  |
| cols-w-xs   | 320px  |
| cols-w-sm   | 384px  |
| cols-w-md   | 448px  |
| cols-w-lg   | 512px  |
| cols-w-xl   | 576px  |
| cols-w-2xl  | 672px  |
| cols-w-3xl  | 768px  |
| cols-w-4xl  | 896px  |
| cols-w-5xl  | 1024px |
| cols-w-6xl  | 1152px |
| cols-w-7xl  | 1280px |

</details>

<details>
<summary>cols-w-[x]</summary>

> <em>Column width has arbitrary value support.</em>

```css
.cols-w-\[x\] {
  --vishesh-columns-width: x;
}
```

</details>

<br />

<Callout type="warning">
  <em>When using both count and width, count functions as the </em> <br />
  <em>maximum column count, while width serves as the minimum </em> <br />
  <em>column width. Count may be fewer than defined, but width </em> <br />
  <em>will not be reduced.</em>
</Callout>

```html

  class="columns cols-2 cols-dw-medium cols-dc-purple-500 cols-gap-4 xl:cols-4 xl:cols-gap-10"
>
  <!-- ... -->
  <img
    class="w-full object-cover rounded-lg"
    src="https://some-website.com/image.png"
  />
  <!-- ... -->
  <img
    class="w-full object-cover rounded-lg"
    src="https://some-website.com/image.png"
  />
  <!-- ... -->
</div>
```
