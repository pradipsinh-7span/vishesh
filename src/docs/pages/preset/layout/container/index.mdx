# Container

In responsive web design, a container is a wrapper element that provides a structured and adaptable layout for content.

<Callout type="warning">
  <em>The Vishesh Preset container is _**`tightly bound to breakpoints`**_.</em>
  <em> So, it only generates container classes for</em>
  <em> breakpoints defined in the configurations.</em>
  <em> [Learn more](/preset/configurations/core-plugins#list-of-plugins)</em>
</Callout>

<br />
<br />

## Types of container

<details open>
<summary>Fixed Container</summary>

A fixed container has a predetermined width that adjusts its size at every breakpoint. This enables the layout to seamlessly adapt to various screen sizes while preserving a consistent width within each breakpoint, ensuring a cohesive and visually appealing design across different devices.

</details>

<details>
<summary>Fluid Container</summary>

A fluid container dynamically adapts its width to fit the available space while simultaneously adjusting its padding at different breakpoints. This ensures that the container scales proportionally, maintains a flexible layout, and retains consistent spacing and alignment.

</details>

<details>
<summary>Twin container</summary>

If you choose either a Fixed or Fluid container, you can use only one of them in your project. However, with the Twin container, you can use both Fixed and Fluid containers together under certain conditions.

</details>

<br />

## **Setup**

```diff
{
# // ...
  theme: {
#   /* Remove Tailwind container configuration */
-   container: {
-     center: true,
-     padding: {
-       DEFAULT: '1rem',
-       sm: '2rem',
-       lg: '4rem',
-       xl: '5rem',
-       '2xl': '6rem',
-     },
-   },
#   // ...
    extend: {
#     /* Default values are already present */
#     /* No need to add them again, provided here just for reference */
+     container: {
+       mode: "fixed",
+       center: true,
+       spacing: {
+         zero: 16,
+         xs: 16,
+         sm: 28,
+         md: 40,
+         lg: 64,
+         xl: 80,
+         mac: 128
+         maxw: 240
+       },
+       innerPadding: {
+         xs: 16,
+         sm: 0,
+         md: 0,
+         lg: 0,
+         xl: 0,
+         mac: 0,
+         maxw: 0
+       }
+     },
#     // ...
    }
  },
# // ...
}
```

## Configuration

The container configurations allow you to customize the behavior and appearance of the container component.

<details>
<summary>mode</summary>

> Determines the container mode

```js
container: {
  mode: "fixed" | "fluid" | "twin",
  // ...
}
```

| Option    | Descrption                                                                               |
| --------- | ---------------------------------------------------------------------------------------- |
| **fixed** | Generates a container with a fixed width that adjusts its size at different breakpoints. |
| **fluid** | Generates a container that dynamically adapts its width to fit the available space.      |
| **twin**  | Use _fixed_ and _fluid_ container together                                               |

</details>

<details>
<summary>spacing</summary>

> Defines the maximum width of the container at different breakpoints.

```js
container: {
  // ...
  spacing: {
    "breakpoint key": number,
    // ...
  },
  // ...
}
```

<Callout type="info">
  <em>For the fixed container, `spacing.zero`</em>
  <em> or for the twin container, `spacing.fixed.zero`</em>
  <em> is always 0, regardless of the value you provide.</em>
</Callout>

</details>

> The configuration below is applicable only in fixed mode.

<details>
<summary>center</summary>

> Determines if the fixed container should be centered

- [Click here to learn more](/preset/layout/container/fixed#center)

</details>
<details>
<summary>innerPadding</summary>

> Equal left and right padding applied to the fixed container at each breakpoint

- [Click here to learn more](/preset/layout/container/fixed#inner-padding)

</details>
