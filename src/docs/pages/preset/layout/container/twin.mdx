# Twin Container

When you select either a _Fixed_ or _Fluid_ container, you can use only one at a time. But what if you wish to use both simultaneously?

Introducing the _Twin container_, which involves some class name adjustments and additional rules.

You can use fixed and fluid container configurations just as you would when the container type is set to fixed or fluid, with some exceptions mentioned below.

<br />

## Spacing

<details>
<summary>Toggle Content</summary>

`#1` Spacing applies to both containers. If you want to override it for _Fixed_ or _Fluid_ containers, you can do so by adding an object with the container type key (`#2` and `#3`).

```js
container: {
  mode: "twin",
  // ...
  spacing: {
    "breakpoint key": number,     // #1
    // ...
    fixed: {
      "breakpoint key": number,   // #2
      // ...
    },
    fluid: {
      "breakpoint key": number,   // #3
      // ...
    }
  },
  // ...
}
```

<Callout type="info">
  <em>The `fixed` and `fluid` objects are optional.</em>
</Callout>
</details>

## Class Names

<details>
<summary>Toggle Content</summary>

<Callout type="error">
  <em>Don't mix Fixed and Fluid container.</em>
</Callout>

<h3>Auto</h3>

For auto classes, there is no change in the name.

1. `container-fixed`
2. `container-fluid`

<h3>Manual</h3>

For manual classes, you need to explicitly add the container type in the class name to identify which container you are using.

| Type  | Before       | After                |
| :---- | :----------- | :------------------- |
| Fixed | container    | container-fixed-base |
|       | container-sm | container-fixed-sm   |
|       | container-md | container-fixed-md   |
|       | ...          | and so on...         |
| Fluid | container    | container-fluid-base |
|       | container-sm | container-fluid-sm   |
|       | container-md | container-fluid-md   |
|       | ...          | and so on...         |

> <em>If `center` is set to false</em>

| Type  | Before           | After                  |
| :---- | :--------------- | :--------------------- |
| Fixed | container-left   | container-fixed-left   |
|       | container-center | container-fixed-center |
|       | container-right  | container-fixed-right  |

</details>

## Reset class

```css
.container-none {
  width: initial;
  margin-left: unset;
  margin-right: unset;
  padding-left: unset;
  padding-right: unset;
  max-width: unset;
  display: initial;
}
```

## Basic usage

<Callout type="error">
  <em>Don't mix Fixed and Fluid container.</em>
</Callout>

<details>
<summary>Fixed and Fluid container on different divs</summary>

<Callout type="error">
  <em>Don't mix Auto and Manual classes.</em>
</Callout>

<h4 class="no-nx">Auto</h4>

> <em>Automatically set max-width at each breakpoint.</em>

```js
class="container-fixed"
```

> <em>Automatically set horizontal padding at each breakpoint.</em>

```js
class="container-fluid"
```

<h4 class="no-nx">Manual</h4>

> <em>Manually set max-width at a specific breakpoint.</em>

```js
class="container-fixed-base container-fixed-md"
```

> <em>Manually set horizontal padding at a specific breakpoint.</em>

```js
class="container-fluid-base container-fluid-md"
```

</details>

<details>
<summary>Fixed and Fluid container on same div</summary>

<Callout type="info">
  <em>You can use Fixed and Fluid container on same div only</em>
  <em> by maintaining CSS specificity.</em>
</Callout>

<Callout type="error">
  <em>Don't mix Auto and Manual classes.</em>
</Callout>

<h4 class="no-nx">Auto</h4>

> <em>
>   Automatically set max-width for Fixed container and horizontal padding for
>   Fluid container at each breakpoint.
> </em>

```js
class="rxl:container-fluid xl:container-fixed"
```

<h4 class="no-nx">Manual</h4>

> <em>
>   Manually set max-width for Fixed container and horizontal padding for Fluid
>   container at specific breakpoint.
> </em>

```js
class="rxl:container-fluid-base md-rxl:container-fluid-md xl:container-fixed-base xl:container-fixed-mac"
```

</details>
