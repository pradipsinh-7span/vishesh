# Fixed container

<Callout type="info">
  max width = breakpoint - ( 2 * spacing ) <br />
  <em>_please refer_ [breakpoints][breakpoints]</em>
</Callout>

<br />

## Max width

#### Auto

> `container-fixed`

| breakpoint | \<375px | ≥375px | ≥640px | ≥768px | ≥1024px | ≥1280  | ≥1440px | ≥1920px |

| spacing    | 0       | 16     | 28     | 40     | 64      | 80     | 128     | 240     |

## Inner padding

Equal left and right padding applied to the container at each breakpoint

#### number

- number should be 0 or greater

#### string

- valid css like "16px", "1rem", etc...first-letter:

#### object

- applied to specific breakpoint
  - key: breakpoint key
  - value: number or string

<br />

## Center

#### true

- horizontally align the container to center
- this is set by default

#### false

- Below classes are available to manually set alignment

| Class            | Description                         |
| :--------------- | :---------------------------------- |
| container-left   | Aligns the container to the left.   |
| container-center | Centers the container horizontally. |
| container-right  | Aligns the container to the right.  |

<br />

### Reset Container

```css
/* Vishesh generates container-none class */
/* to reset the fixed container */

.container-none {
  width: initial;
  margin-left: unset;
  margin-right: unset;
  max-width: unset;
  display: initial;
}
```

<Callout type="warning">
  Please consider [css specificity][specificity] when using `container-none`
</Callout>

<br />

### Basic usage

```js
class="container container-md"
```
